## 2. Screen Layout

An IRS in Murex is displayed under **Financial definition** with two legs side by side. The top section shows:
- `Generator` â€” the template driving leg defaults
- `Maturity` â€” contractual and adjusted maturity dates
- `Nominal` â€” the notional amount
- `Currency` â€” the single currency of the swap (same on both legs)

Each leg panel shows:
- Rate type: `Fixed rate` or `Floating rate`
- `Index` â€” floating rate index (floating leg only)
- `Currency`
- `Start date` / `Maturity`
- `Denomination` checkbox â€” if checked, flows shown in denomination units
- `Frequency` â€” payment frequency (fixed leg only; floating leg uses index tenor)
- `Convention` â€” day count convention
- `Rate` (fixed leg) or `1st fixing` + `Margin` (floating leg)

For **compounding swaps**, an additional field appears in the leg configuration:
- `Compounding` â€” the compounding rule applied (e.g. `(I+M) at I`)

The **Exotic** and **Leg characteristics** tabs contain further configuration (discounting curves, amortization schedule).

## 4. Flow Types

IRS flows are displayed in the **Flow schedule** screen:

| Code | Description |
|------|-------------|
| `INT` | Periodic interest payment â€” will be paid on the payment date |
| `INC` | Incremental interest flow (compounding swaps only) â€” not paid immediately; compounded with the next period and paid at the end of the compounding window |
| `FLW` | Additional flow â€” generated by certain indices with settlement lags (e.g. GBPLIBOR) |
| `PRI` | Capital flow exchanged |
| `INI` | Initial capital flow (notional record â€” not exchanged for mono-currency IRS) |
| `FIN` | Final capital flow (notional record â€” not exchanged for mono-currency IRS) |
| `AMO` | Amortized capital flow |

> **Identifying a compounding leg in Murex:** the presence of `INC` flows in the flow schedule confirms that leg is the compounding leg. `INC` flows appear in italic in some Murex versions, indicating they will not be paid individually.

Flow schedule columns:

| Column | Description |
|--------|-------------|
| `Sub. Tp` | Flow subtype (`INT`, `INC`, `FLW`, `AMO`) |
| `Calculation Start date` | Period calculation start |
| `Calculation End date` | Period calculation end |
| `Remaining Capital` | Remaining notional for this period (shown in italic if customized amortization) |
| `Fixing Date` | Date on which the floating rate was fixed |
| `Rate` | The fixed or floating rate applied |
| `Margin` | Additive spread over the rate |
| `Rate Factor` | Multiplier (1.0000 standard) |
| `Payment Date` | Date the flow is paid (blank for `INC` flows) |
| `Flow` | Calculated amount |
| `Cur` | Currency |

## 5. Compounding Swap Configuration in Murex

### Enabling compounding: Frequency Ratio
Compounding is triggered by setting the **Payment schedule Frequency ratio** to a value greater than 1 in the Schedule Definition screen of the compounding leg. A ratio of 2 means one payment is made every two calculation periods.

Schedule Definition fields:

| Field | Description |
|-------|-------------|
| `Calculation start schedule` | `Driving schedule` (drives the calculation period dates) |
| `Schedule generator` | Generator for calculation dates (e.g. `3M MODFOL EOM`) |
| `Calculation end schedule` | `Equal to` (= start schedule) |
| `Payment schedule` | `Equal to` (= calculation start schedule) |
| `Frequency ratio` | Integer > 1 to enable compounding; direction flag appears alongside |
| `Fixing schedule` | `Deduced from` calculation start schedule |
| `Deduction formula` | e.g. `-2 OPEN DAYS` (fixing lag) |
| `Calculations follow fixings` | `No (time series)` |
| `Capital schedules` | `No` |
| `Single period` | `No` (for multi-period compounding) |
| `First fixing calendar` | `Inherited` |

### Compounding direction and stub handling
When Frequency ratio > 1, a direction flag appears:

| Flag | Direction | Stub treatment |
|------|-----------|----------------|
| `Backward (broken periods included)` | From maturity backward | Stub included in compounding |
| `Backward (broken periods excluded)` | From maturity backward | Stub excluded â€” paid separately |
| `Forward (broken periods included)` | From start date forward | Stub included in compounding |
| `Forward (broken periods excluded)` | From start date forward | Stub excluded â€” paid separately |

### Compounding rule (Compounding mode field)
After setting the frequency ratio, the `Compounding` field in the leg panel specifies how margin and rate interact:

| Rule | Description |
|------|-------------|
| `(I+M) at (I+M)` | Flow including margin compounded at rate including margin |
| `(I+M) at I` | Flow including margin compounded at rate excluding margin |
| `(I at I) + M` | Flow excluding margin compounded; margin paid separately |
| `I at (I+M)` | Flow excluding margin compounded at rate including margin |
| `I at (I+M) + M` | Flow excluding margin compounded at rate including margin; margin added separately |
| `No compounding` | Flows simply summed |
| `(I+M1) at (I+M2)` | Two separate margins applied to flow and to compounding rate |
| `(I+M) at C` | Non-linear: flow compounded using an option value function f(r, m, K) |

## 6. Cash Flow Calculation

### Standard interest flow formula (all IRS types)
```
Flow = Notional Ă— (Rate / 100) Ă— DayFrac
DayFrac = Days / Base
Days = Calculation End date âˆ’ Calculation Start date (calendar days)
Base = Day count denominator (365 for ACT/365, 360 for ACT/360)
```

### Example â€” Vanilla IRS fixed leg (GBP, ACT/365)
```
Notional:   93,750,000 GBP
Start date: 31/12/2014
End date:   30/06/2015
Days:       181
Base:       365
DayFrac:    0.495890
Rate:       3.6200% (customized for this period)
Flow:       93,750,000 Ă— (3.62/100) Ă— 0.495890 = 1,682,928.08 GBP
```

### Example â€” Basis swap floating leg with margin (EUR, ACT/360)
```
Notional:   4,400,000 EUR
Start date: 06/11/2018
End date:   06/05/2019
Days:       181
Base:       360
DayFrac:    0.502778
Rate:       -0.3646% (EUR EONIA COMP fixing)
Margin:     +0.15%
Flow:       4,400,000 Ă— ((-0.3646 + 0.15) / 100) Ă— 0.502778 = -4,746.64 EUR
```

### Example â€” Compounding swap: calculating a compounded payment (USLIBOR3M vs USLIBOR6M)

The USLIBOR3M leg uses compounding (frequency ratio = 2) to align with the semi-annual payment schedule of the USLIBOR6M leg. Two quarterly `INC` flows are compounded into one semi-annual `INT` payment.

**Step 1 â€” Calculate Flow 1 (INC, period 31/12/2019 â†’ 31/03/2020):**
```
Remaining capital: 30,055,961.16 USD
Start:  31/12/2019 | End: 31/03/2020 | Days: 91 | Base: 360 | DayFrac: 0.252778
Rate:   1.94463% | Margin: 0.111%
Flow 1: 30,055,961.16 Ă— ((1.94463 + 0.111) / 100) Ă— 0.252778 = 156,176.06 USD  [INC â€” not paid]
```

**Step 2 â€” Calculate Flow 2 (INT, period 31/03/2020 â†’ 30/06/2020):**
```
Remaining capital: 30,055,961.16 USD
Start:  31/03/2020 | End: 30/06/2020 | Days: 91 | Base: 360 | DayFrac: 0.252778
Rate:   1.45013% | Margin: 0.111%
Rate real: ((1.45013 + 0.111) / 100) Ă— 0.252778 = 0.003946
Flow 2: 30,055,961.16 Ă— 0.003946 = 118,606.53 USD
```

**Step 3 â€” Compound Flow 1 using the estimation curve:**
The CF curve factor is derived from the estimation curve (USD_DEPOSWAP_3M) interpolated at the fixing date (27 Mar):
```
Rate curva: 1.478404%
BASE curva: 365
CF curva: (1 + (1.478404 / 100))^(91/365) = 1.003665606
Flow 1 compounded: 156,176.06 Ă— 1.003665606 = 156,748.54 USD
```

**Step 4 â€” Total payment on 30/06/2020:**
```
Flow 1 compounded + Flow 2 = 156,748.54 + 118,606.53 = 275,355.06 USD
```

This total is the single `INT` payment on 30 Jun 2020, covering both quarterly periods.

## 7. Pricing and Curves

To view or change the discounting and estimation curves for a specific trade:
**Deal Pricing â†’ Market data tab**

- `Discounting rate curve` â€” curve used to discount flows on each leg
- `Estimation rate curve` â€” curve used to project future floating rates (floating leg only)

**For compounding swaps:** the estimation curve is also used to calculate the compounding factor (CF curva) applied to `INC` flows. The interpolator must be accessed at the fixing date of the relevant period.

**Murex paths:**
- Rate and discount curves: `Market Data / Rates / Revaluation Rate Curves / Select Currency / Select Curve`
- Lambda compounded: `Historical Data / Rates / O/N financing rates / Select Currency / Select Start Date`
- Rate interpolator: accessible from the curve screen â€” use the fixing date as the horizon

## 8. Customizations

IRS flows can be customized at trade level (overriding generator defaults). Common customizations:
- **Custom rate for specific periods** â€” a specific fixing rate stored on a flow instead of the live index rate (visible in italic in the flow schedule)
- **Custom amortization schedule** â€” non-standard notional reduction profile (flows in italic/blue in the flow schedule)
